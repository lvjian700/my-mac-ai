# Config Support

## Context

Add configuration for the default account and calendar used by `ical add`.

The CLI needs two config levels:

- Local config for project- or directory-specific defaults.
- User config for global personal defaults shared across local contexts.

This lets a user keep a common default account/calendar while still allowing a specific local context to override it.

Common use cases:

- Advanced users explicitly ask to change the default account/calendar.
- Events are being created in a local calendar, but the user wants future events to sync through an iCloud account.

## Design

Storage:

- Local config: `./.ical/config.json`
- User config: `~/.my-mac-ai/ical/config.json`

Precedence for `ical add`:

1. Explicit CLI flags: `--account` and `--calendar`
2. Nearest ancestor local config
3. User config
4. EventKit default calendar

Write behavior:

- `ical config add` writes local config by default.
- `ical config add --user` writes user-global config.
- `ical config add` with no values is read-only and shows the effective add config.
- Local config lookup walks up from the current working directory and uses the nearest `.ical/config.json`.
- Config files are JSON and currently store add-event defaults under the `add` key.

## Commands

- `ical config` - view user, local, and effective config.
- `ical config add` - view the effective config used by `ical add`.
- `ical config add --account <account> --calendar <calendar>` - set local defaults for `ical add`.
- `ical config add --user --account <account> --calendar <calendar>` - set user-global defaults for `ical add`.

## iCloud Sync Workflow

When a user says new events are appearing only locally or not syncing to iCloud:

1. Run `ical list --format json` to find the exact iCloud account and calendar names.
2. Run `ical config add --user --account <icloud-account> --calendar <calendar>` to make future `ical add` commands use that iCloud calendar globally.
3. Use local config instead, without `--user`, only when the default should apply to the current project/directory.
4. For one-off events, prefer explicit `ical add --account <account> --calendar <calendar>` instead of changing defaults.

## Implementation Plan

- Add `ConfigStore` for JSON read/write, nearest-local discovery, and effective config merging.
- Add `ConfigCommand` and `ConfigAddCommand` for viewing and updating config.
- Update `AddEventCommand` so config supplies account/calendar only when the matching CLI flag is omitted.
- Add formatter support for text and JSON config output.
- Keep existing EventKit calendar resolution behavior unchanged after the effective account/calendar is resolved.
