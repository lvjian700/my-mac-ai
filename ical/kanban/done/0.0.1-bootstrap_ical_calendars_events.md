# 1 — Bootstrap

Bootstrap a Swift 6.3 CLI app exposing Apple Calendar via EventKit.

## Decisions

- **No reminders** — calendar only
- **Flat command structure** — no `calendar` subgroup, commands live directly under `ical`
- **No delete** — deferred
- **macOS .v14 minimum** — required for `requestFullAccessToEvents()` async API
- **`main.swift` entry point** — avoids `@main`/`main.swift` conflict with ArgumentParser
- **`EventKitService` is `@MainActor`** — `AsyncParsableCommand` auto-hops to main actor, satisfying EventKit threading without extra boilerplate

## Files created

| File | Purpose |
|------|---------|
| `Package.swift` | ArgumentParser dep, macOS .v14, EventKit linker, Info.plist embedded via `-sectcreate` linker flag |
| `Sources/ical/main.swift` | Entry point: `IcalCommand.main()` |
| `Sources/ical/IcalCommand.swift` | Root `ParsableCommand`, registers subcommands |
| `Sources/ical/EventKitService.swift` | `@MainActor` singleton — access request, list calendars, list/add events |
| `Sources/ical/OutputFormatter.swift` | `OutputFormat` enum, `DateParser`, text/JSON printers for calendars and events |
| `Sources/ical/Resources/Info.plist` | `NSCalendarsFullAccessUsageDescription` for permission prompt |
| `Sources/ical/Commands/Calendar/ListCalendarsCommand.swift` | `ical list` |
| `Sources/ical/Commands/Calendar/ListEventsCommand.swift` | `ical events` |
| `Sources/ical/Commands/Calendar/AddEventCommand.swift` | `ical add` |

## Build fix

`Info.plist` cannot be a `.process()` resource in an executable target (SwiftPM rejects it). Embedded instead via `-sectcreate __TEXT __info_plist` linker flag in `Package.swift`. `ListCalendarsCommand` also had stale API calls (`requestEventsAccess` → `requestAccess`, `listCalendars(for:)` → `listCalendars()`) fixed alongside adding `@MainActor` to `run()`.
