# Add `ical update` Command

## Context

The `ical` CLI had no way to edit an existing event. Users who created an event with a wrong title or time had to open the Calendar app to fix it. This feature adds `ical update <event-id>` following the existing patterns.

Event IDs are obtained via `ical events --format json` (the `id` field in each event).

## Changes

### New command: `ical update <event-id>`

Options:
- `--title` — new event title
- `--start` — new start date/time (ISO-8601 or YYYY-MM-DD)
- `--end` — absolute end date/time (overrides duration preservation)
- `--short` / `--normal` / `--long` — duration shortcuts (15/30/45 min), take priority over `--end`
- `--calendar` — move event to a different calendar
- `--account` — disambiguate calendars with duplicate names
- `--format` — text or json

Key behaviour:
- Only provided fields are changed; omitted fields are untouched
- `--start` alone (no `--end` / duration flag): moves the event, **preserving original duration**
- No options provided: warns to stderr, prints current event, exits 0
- Recurring events: `span: .thisEvent` — only the specific occurrence is updated

### Shared end-date resolution

Extracted `resolveFlaggedEndDate(from:short:long:normal:endStr:)` free function shared by both `AddEventCommand` and `UpdateEventCommand`, ensuring consistent flag ordering and behaviour.

### EventKitService additions

- `fetchEvent(byId:)` — fetch event by `eventIdentifier`; throws `IcalError.eventNotFound(id:)` if missing
- `updateEvent(id:title:startDate:endDate:calendarName:account:)` — mutate and save with `span: .thisEvent`
- `IcalError.eventNotFound(id:)` — new error case

### Validation & safety

- Guard against IUO crash on `EKEvent.startDate`/`.endDate` (declared as `Date!`)
- End > start validation in both `add` and `update`

### Tests (27 total, +12 new)

New `UpdateEventCommandTests` covering:
- Argument parsing (all fields nil/false by default)
- `resolveEndDate` for all paths: short/long/normal flags, explicit `--end`, `--start`-only duration preservation, nil-return (no change)
- `--short` overrides explicit `--end`
- `--calendar`-only recognised as a change
- `IcalError.eventNotFound` error message content
